### 修复微信开发者工具编译错误

**问题**：
- `Undefined variable` 报错（多个 SCSS 文件）
- `app.json rendererOptions.skyline["tagNameStyleIsolation"]` 无效字段警告

**根因**：
1. `_tokens.scss`（partial 文件）里**漏定义了** `$on-tertiary-fixed` 和 `$on-tertiary-fixed-variant`，被 `analysis.scss` `.tip-num` 引用
2. `$safe-bottom: env(safe-area-inset-bottom);` 旧版 dart-sass 在 SCSS 变量赋值时不识别 CSS `env()` 函数
3. `tagNameStyleIsolation` 是 3.x 已被移除的 Skyline 配置

**修复**：
- 补全 `$on-tertiary-fixed: #271900;` 与 `$on-tertiary-fixed-variant: #5e4200;`
- 把 `$safe-bottom` 改成纯 rpx 数字 0rpx（运行时通过 `wx.getWindowInfo().safeArea` 算）
- 删除 `app.json` 里 `tagNameStyleIsolation` 字段
- 把 `_tokens.scss` 重命名为 `tokens.scss`（非 partial），更新所有 `@import` 路径

**教训**：写 WXSS token 时一定要对照原 HTML 完整列出所有 Material 3 token（on-* / *-fixed / *-container 系列），不能漏。

### 修复首页"显示不对"问题

**问题**：小程序三个页面顶栏都把下方内容遮住，搜索框和首屏内容被压在顶栏后面。

**根因**：
1. 顶栏用 `position: sticky; top: 0;` —— sticky 不脱离文档流，但页面 `.page` 整体只 `padding-top: {{statusBarHeight}}px`（≈20px），没给顶栏 88rpx (44px) 高度留出空间
2. SCSS 错误写法 `padding-top: {{statusBarHeight}}px + 88rpx` —— `{{}}` 是 WXML 模板语法，**SCSS 里写就只是字符**，不会编译报错但不会执行。SCSS 只能编译纯 SCSS 表达式
3. 顶栏应该 `position: fixed`（脱离文档流），同时 content 用 inline style 动态算 padding-top 避让

**修复方案**（3 个页面通用）：
- `topbar` 改成 `position: fixed; top: {{statusBarHeight}}px;`（WXML inline 传 px）
- 在 ts 的 onLoad 计算 `contentTop` = `(statusBarHeight + 44 + 8) * 2` rpx（×2 是 px → rpx 换算）
- content 容器用 `<view class="content" style="padding-top: {{contentTop}}rpx;">`
- 同步 padding-bottom 200 → 240rpx（给 FAB 留空间）

**顺手修复**：
- 推荐卡宽度 620 → 638rpx（85% × 750）
- 推荐卡高度 388 → 400rpx（更接近 16:10）
- section gap 24rpx → 16rpx（对齐原 HTML space-y-md=16px）
- Bento gap 24 → 16rpx
- Recipe list gap 24 → 16rpx
- FAB 位置 right 48 → 24rpx（对齐原 HTML right-6）
- 搜索框高度 96 → 88rpx

**教训**：
- 永远记住：SCSS 只编译 SCSS 表达式，`{{}}` 在 SCSS 里就是普通字符。要插值用 inline style 传
- 顶栏 fixed 后必须在主体加 padding-top 避让，否则一定遮内容
- px → rpx 换算固定 ×2（小程序 rpx 体系：屏幕宽 750rpx）
- 写小程序的"避让"高度时用 ts 计算然后 setData，比在 SCSS 里硬编更鲁棒（适配不同机型状态栏高度）
